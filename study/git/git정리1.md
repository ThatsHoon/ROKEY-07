**🚀 Git & GitHub 핵심 요약 가이드**

## **1\. 기본 작업 흐름 (The 3-Step Cycle)**

Git 작업은 크게 \*\*'장바구니 담기 \-\> 라벨 붙이기 \-\> 트럭에 싣기'\*\*의 과정을 반복합니다.

| 단계 | 명령어 | 설명 |
| :---- | :---- | :---- |
| **1\. 스테이징** | git add . | 현재 변경된 모든 파일을 장바구니에 담음 (일회성) |
| **2\. 커밋** | git commit \-m "메시지" | 작업을 기록으로 남김 (내 컴퓨터에 저장) |
| **3\. 푸시** | git push origin main | 내 기록을 GitHub(원격 저장소)로 업로드 |

## ---

**2\. 주요 용어 정리**

* **origin**: 연결된 GitHub 저장소 주소의 **별명**.  
* **main**: 프로젝트의 중심이 되는 **기본 줄기(브랜치)** 이름.  
* **HEAD**: 내가 지금 보고 있는 **현재 시점**을 가리키는 포인터.

## ---

**3\. 브랜치(Branch)와 이동**

복잡한 커밋 해시(ID) 대신 직관적인 이름을 사용하여 작업 단위를 나눕니다.

* **목록 확인**: git branch (로컬), git branch \-a (전체)  
* **생성**: git branch \[이름\]  
* **이동**: git switch \[이름\] (또는 git checkout \[이름\])  
* **강제 이동**: git branch \-f \[이름\] \[커밋해시\] (특정 커밋으로 브랜치 말뚝 박기)

## ---

**4\. 기록 확인 (Log)**

* **간결하게 보기**: git log \--oneline  
* **그래프로 보기**: git log \--oneline \--graph \--all  
* **모든 행적 확인**: git reflog (실수로 지운 커밋을 찾을 때 유용한 '최후의 보루')

## ---

**5\. 되돌리기 (Undo)**

상황에 따라 '기록을 남기며 취소'할지, '아예 시간을 되돌릴지' 선택합니다.

### **Revert (안전한 취소)**

* 명령어: git revert \[커밋해시\]  
* 특징: 기존 기록은 유지하고, 이를 **반대로 수행하는 새로운 커밋**을 추가함. 푸시 이후에 사용 권장.

### **Reset (강력한 되돌리기)**

* 명령어: git reset \--hard \[커밋해시\]  
* 특징: 지정한 시점으로 **시간을 완전히 되돌림**. 이후의 기록은 사라짐.  
* **주의**: 이미 푸시된 브랜치를 리셋했다면 git push \-f (강제 푸시)가 필요함.

## ---

**6\. 태그 (Tag)**

중요한 시점(예: 버전 출시)에 고정된 이름표를 붙입니다.

* **태그 달기**: git tag \-a v1.0 \-m "설명"  
* **과거에 달기**: git tag v0.9 \[커밋해시\]  
* **GitHub 전송**: git push origin \[태그명\]

## ---

**7\. 응급 처치 가이드**

### **Vim 편집기 탈출 (검은 화면)**

1. Esc 키 누르기  
2. :wq 입력 후 엔터 (저장 후 종료)  
3. :q\! 입력 후 엔터 (저장하지 않고 종료)

### **Detached HEAD (유령 상태)**

브랜치 이름이 아닌 커밋 ID로 이동했을 때 발생합니다.

* **해결**: git switch main으로 돌아오거나, git switch \-c \[새이름\]으로 새 브랜치를 만드세요.

### ---

**Rebase (줄기 정리)**

* 내 작업을 최신 메인 줄기 위로 재배치하여 히스토리를 깔끔하게 한 줄로 만듭니다.  
* **주의**: 내 컴퓨터에만 있는 커밋들에 사용하는 것이 안전합니다.
